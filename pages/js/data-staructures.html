Sets

The Set object lets you store unique values of any type, whether primitive values or object references.

Set has no indexes (no way to get item from the set) 

                                
setto[0]; // undefined,

Looping like other iterables 

                                
for (const order of setto) {
  console.log(order);
}

Sets fiddle

Set properties - writeable

Defining Set and set properties 

                                
const setto = new Set(['Pasta', 'Pizza', 'Pizza', 'Risotto', 'Pasta', 'Pizza']);
console.log(setto); // {'Pasta', 'Pizza', 'Risotto'}

new Set('Artem'); // {'A', 'r', 't', 'e', 'm'}; // strings also iterable

add()

Inserts a new element with a specified value in to a Set object, if there isn't an element with the same value already in the Set.

Returns the Set object with added value. 

                                
add(value);

setto.add('Garlic Bread');

delete()

Removes a specified value from this set, if it is in the set.

Returns true if value was already in Set; otherwise false. 

                                
delete(value);

setto.delete('Risotto');

clear()

Removes all elements from the Set object. 

                                
setto.clear(); // none (undefined)

Set properties - read-only

size

                                
setto.size; // 3

has()

a boolean asserting whether an element is present with the given value in the Set object or not. 

                                
setto.has('Pizza'); // true
setto.has('Bread'); // false

keys()

An alias for the values() method.

Return a new iterable iterator object. 

                                
setto.keys();

setto.next().value;

values()

Returns a new iterator object that yields the values for each element in the Set object in insertion order. 

                                
setto.values();

setto.next().value;

entries()

Returns a new iterator object that contains an array of [value, value] for each element in the Set object, in insertion order. This is similar to the Map object, so that each entry's key is the same as its value for a Set. 

                                
setto.entries();

setto.next().value;

forEach()

Calls callbackFn once for each value present in the Set object, in insertion order. If a thisArg parameter is provided, it will be used as the this value for each invocation of callbackFn. 

                                
forEach(callbackFn, thisArg)

setto.forEach((key, value, set) => console.log(key, value, set)); // key and value are the same in Sets

"New" methods

union()

Takes a set and returns a new set containing elements which are in either or both of this set and the given set. 

                                
union(other)

const evens = new Set([2, 4, 6, 8]);
const squares = new Set([1, 4, 9]);
evens.union(squares); // Set(6) { 2, 4, 6, 8, 1, 9 }

difference()

Takes a set and returns a new set containing elements in this set but not in the given set. 

                                
difference(other)

const odds = new Set([1, 3, 5, 7, 9]);
const squares = new Set([1, 4, 9]);
odds.difference(squares); // Set(3) { 3, 5, 7 }

intersection()

Takes a set and returns a new set containing elements in both this set and the given set. 

                                
intersection(other)

const odds = new Set([1, 3, 5, 7, 9]);
const squares = new Set([1, 4, 9]);
odds.intersection(squares); // Set(2) { 1, 9 }

isSubsetOf()

Takes a set and returns a boolean indicating if all elements of this set are in the given set. 

                                
isSubsetOf(other)

const fours = new Set([4, 8, 12, 16]);
const evens = new Set([2, 4, 6, 8, 10, 12, 14, 16, 18]);
fours.isSubsetOf(evens);    // true

const odds = new Set([3, 5, 7, 9, 11, 13, 15, 17, 19]);
const primes = new Set([2, 3, 5, 7, 11, 13, 17, 19]);
odds.isSupersetOf(primes);  // false (primes not contains 9, 15)

isSupersetOf()

Takes a set and returns a boolean indicating if all elements of the given set are in this set. 

                                
isSubsetOf(other)

const evens = new Set([2, 4, 6, 8, 10, 12, 14, 16, 18]);
const fours = new Set([4, 8, 12, 16]);
evens.isSupersetOf(fours);             // true (all elements in this contains in given set)

const odds = new Set([3, 5, 7, 9, 11, 13, 15, 17, 19]);
const primes = new Set([2, 3, 5, 7, 11, 13, 17, 19]);
odds.isSupersetOf(primes);             // false (odds not contains 2)

symmetricDifference()

Takes a set and returns a new set containing elements which are in either this set or the given set, but not in both. 

                                
symmetricDifference(other)

const evens = new Set([2, 4, 6, 8]);
const squares = new Set([1, 4, 9]);
evens.symmetricDifference(squares); // Set(5) { 2, 6, 8, 1, 9 }

isDisjointFrom()

Takes a set and returns a boolean indicating if this set has no elements in common with the given set. 

                                
isDisjointFrom(other)

const primes = new Set([2, 3, 5, 7, 11, 13, 17, 19]);
const squares = new Set([1, 4, 9, 16]);
primes.isDisjointFrom(squares); // true

Some examples

                                
const staff = ['Waiter', 'Chef', 'Waiter', 'Manager', 'Chef', 'Waiter'];
const staffUnique = [...new Set(staff)]; // ['Waiter', 'Chef', 'Manager']

// how many unique positions
new Set(staff).size;        // 3
new Set('Artem').size;      // 5
new Set('Halloween').size;  // 7

Maps

The Map object holds key-value pairs and remembers the original insertion order of the keys. Any value (both objects and primitive values) may be used as either a key or a value.

Maps fiddle

Map properties - writeable

Defining Map and set properties 

                                
const mappo = new Map([[key, value], [key, value]]);

const mappi = new Map();
mappi.set('name', 'Classico Italiano');
mappi.set(1, 'Firenze, Italy');
mappi.set(2, 'Lisbon, Portugal');
console.log(mappi); // {'name' => 'Classico Italiano', 1 => 'Firenze, Italy', 2 => 'Lisbon, Portugal'}

set()

Sets the value for the passed key in the Map object. Returns the Map object. 

                                
mappi.set(key, value);

mappi
  .set('categories', ['Italian', 'Pizzeria', 'Vegetarian', 'Organic'])
  .set('open', 11)
  .set('close', 23)
  .set(true, 'We are open')
  .set(false, 'We are closed');

delete()

Returns true if an element in the Map object existed and has been removed, or false if the element does not exist. map.has(key) will return false afterwards. 

                                
mappi.delete(key); // true or false

clear()

Removes all key-value pairs from the Map object.

                                
mappi.clear(); // none (undefined)

Maps properties - read-only

size

Returns the number of key/value pairs in the Map object. 

                                
mappi.size; // 3

get()

                                
mappi.get('name');  // Classico Italiano
mappi.get(true);    // We are open

mappi.get('1');     // undefined
mappi.get(1);       // Firenze, Italy

const time = 8;
mappi.get(time > mappi.get('open') && time < mappi.get('close')); // We are closed

has()

Returns a boolean indicating whether a value has been associated with the passed key in the Map object or not. 

                                
mappi.has(key)
[...mappi.values()].includes('value')           // check for values
[...mappo.entries()].flat().includes('Artem')   // check for values

mappi.has('categories'); // true

keys()

Returns a new map iterator object that contains the keys for each element in this map in insertion order. 

                                
mappi.keys(); // { key, key, key, key }

mappi.next().value;

values()

Returns a new map iterator object that contains the values for each element in this map in insertion order. 

                                
mappi.values(); // { value, value, value, value }

mappi.next().value;

entries()

Returns a new map iterator object that contains the [key, value] pairs for each element in this map in insertion order. 

                                
mappi.entries(); { key => value, key => value }

mappi.next().value;

forEach()

Executes a provided function once per each key/value pair in this map, in insertion order. 

                                
forEach(callbackFn, thisArg)

mappi.forEach((value, key, map) => console.log(`${key} : ${value}`))

Don`t forgot about HEAP

                            
mappi.set([1, 2], 'Test');
mappi.get([1, 2]); // undefined (because it`s different array in the heap, calling function is an expression)

const arr = [1, 2];
mappi.set(arr, 'Test2');
mappi.get(arr); // Test2 (because it`s the same array in the heap)

// DOM elements also can be keys
mappi.set(document.querySelector('h1'), 'heading');
mappi.has(document.querySelector('h1')); // true

Maps: iteration, map converting

Maps iteration fiddle

Iterating maps

                                
const question = new Map([
  ['question', 'What is the best programming language in the world?'],
  [1, 'C'],
  [2, 'Java'],
  [3, 'JavaScript'],
  [4, 'Python'],
  [5, 'C++'],
  ['correct', 3],
  [true, 'Correct 🎉'],
  [false, 'Try again...'],
]);

for (const [key, value] of question) {
  if (typeof key === 'number') {
    console.log(`Answer ${key}: ${value}`)
  };
}

const answer = Number(prompt('Your answer'));
console.log(answer);

// if (answer === question.get('correct')) console.log(question.get(true));
console.log(question.get(answer === question.get('correct')));

Converting

Object to map

Map structure is the same as Object.entries() returns for iterating 

                                
const openingHours = {
  thu: {
    open: 12,
    close: 22,
  },
  fri: {
    open: 11,
    close: 23,
  },
  sat: {
    open: 0, // Open 24 hours
    close: 24,
  },
};

const hoursMap = new Map(Object.entries(openingHours)); { thu => {open, close}, fri => {open, close}, sat => {open, close} }
const hoursFromObject = Object.entries(openingHours); // [ Array(2), Array(2), Array(2) ]

Map to object

                                
const objectFromMap = Object.fromEntries(mappo.entries());

Array to map

                                
const mappo = new Map([[key, value], [key, value]])

Convert map to array

                                
[...question]; // [[Array(2)], [Array(2)], etc.]

Data structures summary

Summary: which data structure to use?

Sources of data

  1. From the program itself:

    • data written directly in source code (e.g. status messages)

    • data what will be displayed on webpage on user actions

  2. From the UI:

    • data input from the user or data written in DOM (e.g. tasks in todo app, expanses in budget app)

  3. From external sources:

    • data fetched for example from web API (Application Programming Interface) (e.g. recipe objects)

Collection data structures: array, object, set, map, weakset, weakmap

Collection data structures that JS does not have: stacks, queues, linked lists, trees, hash tables

How to choose one?

  1. Simple list?

    • array

    • set

  2. Key value pairs?

    • keys allow us to describe values

    • object

    • map

Sets vs Arrays

Arrays

  • use when you need ordered list of values (might contain duplicates)

  • use when you need to manipulate data

Sets

  • use when you need to work with unique values

  • use when high-performance is really important

  • use to remove duplicates from arrays

Objects vs Maps

Objects

  • more "traditional" key/value store ("abused" objects)

  • easier to write and access values with . and []

  • use when you need to include functions (methods)

  • use when working with JSON (can convert to map)

Maps

  • better performance

  • keys can have any data type

  • easy to iterate

  • easy to compute size

  • use when you simply need to map key to values

  • use when you need keys that are not strings
