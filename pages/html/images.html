Tech

Technical details

To account for high-res screens, make image dimensions 2x as big as their displayed size

  • If image have website size 300x300, he`s original size need to be 600x600

  • Scale factor: Actual pixels the screen contains / Pixels represented on screen

  • On high-res screens, scale factor is 2x or even 3x, on "normal" screens it`s just 1x (1 physical pixel = 1 design pixel)

Compress images for a lower file size and better performance

When using multiple images side-by-side, make sure they have the exact same dimensions

Always declare images size

Performance

Fetchpriority

Use fetchpriority and loading="lazy" for better loading performance.

loading eager - default

                            
<img
    src="image.jpg"
    alt="A description of the image."
    height="800"
    width="1200"
    loading="eager" / loading="lazy"
    fetchpriority="high">

Decoding: async or sync.

Use the sync value if the image itself is the most important piece of content to prioritize. 

                            
<img
    src="image.png"
    alt="A description of the image."
    width="300"
    height="200"
    loading="lazy"
    decoding="async" / decoding="sync">

Performance

From GIF, JPG, PNG and AVIF - better use AVIF. With the same look, it`s 30% smaller.

Always use max-inline-size: 100%; and block-size: auto; on images, width dimension in html. Browser use html dimensions for calculating aspect-ration of images, for better performance.

Use custom hotkey in vs code, emmet - update image size.

Preload images what used in js or background css.

                            
<link rel="preload" href="hero.jpg" as="image" fetchpriority="high">

Usage

Default styles

                            
picture {
    display: inline-block;
}

img,
video,
iframe {
    display: block;
    max-inline-size: 100%;
    block-size: auto;
}

Picture element

                                
    <picture>
        <source srcset="full.jpg" media="(min-width: 75em)" width="1200" height="500">
        <source srcset="regular.jpg" media="(min-width: 50em)" width="800" height="400">
        <img src="cropped.jpg" alt="A description of the image." width="400" height="400">
    </picture>

Multiple image formats with a pixel density descriptor 

                            
<picture>
    <source srcset="image.avif 1x, image.avif 2x" type="image/avif">
    <source srcset="image.webp 1x, image.webp 2x" type="image/webp">
    <img src="image.jpg" srcset="image.jpg 1x, image.jpg 2x" alt="descr" width="400" height="400">
</picture>

Sizes and srcset

                            
<img
    srcset="small-image.png 300w, medium-image.png 600w, large-image.png 1200w"
    sizes="(min-width: 66em) 33vw, (min-width: 44em) 50vw, 100vw">

Background image-set

                            
background-image: image-set(
    url("image-2x.webp") type("image/webp") 2x,
    url("image-1x.webp") type("image/webp") 1x,
    url("image-2x.avif") type("image/avif") 2x,
    url("image-1x.avif") type("image/avif") 1x,
    url("image-2x.jpeg") 2x,
    url("image-1x.jpeg") 1x
);
