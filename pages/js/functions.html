Basics

DRY PRINCIPLE: Don't Repeat Yourself (DRY) is a coding principle that encourages writing code that is easier to read, understand, and maintain.

What is function?

Functions allow us to group related code together and reuse it.

Function is a block of code that performs a specific task. 

                                
function logger(message) {
  console.log(`My name is Artem`)
}

Invoking, running or calling a function.

                                
logger()

Declaration vs Expression

Function declaration

Function declaration can be used before they are defined, this process called hoisting.

Function exits immediately after returning the result. 

                                
function calcAge(birthYear) {
  return 2024 - birthYear
}

const age = calcAge(1997)

Function expression

Function expression produce a value to be stored in a variable.

Anonymous function is a function without a name.

                                
const calcAge = function (birthYear) {
  return 2024 - birthYear
}
const age = calcAge2(1997)
console.log(age)

Better use function expression for better readability and reusability.

Arrow functions

Arrow functions are a shorter syntax for writing function expressions.

Arrow functions returns the result of the expression inside the function automatically if the function has no brackets. 

                                
const calcAge = birthYear => 2024 - birthYear
const age = calcAge(1997)
console.log(age)

Arrow function with multiple parameters and a return statement.

Need return keyword to return a value if the function has more than one line. 

                                
const yearsUntilRetirement = (birthYear, firstName) => {
  const age = 2024 - birthYear
  const retirement = 65 - age
  return `${firstName} retires in ${retirement} years`
}

Arrow function has a differences from regular function expressions, they have no 'this' keyword.

About functions

  • Function has a name, parameters, and a body

  • Parameters: placeholders for data that will be provided when the function is called, like local variables in functions

  • Arguments: actual values of function parameters, to provide data to the function

  • Body: block of code that performs a specific task that we want to reuse. Processes the data and returns the result

  • Return statement: exits the function and sends a value back to the caller

  • Calling, running or invoking a function, using parentheses and passing the required arguments

  • Variable to save returned values (function output).

Default values are expressions. We can use parameters that are declared before they called.

Specify undefined, when we have no values for parameter or want to skip, it gonna be changed to default value.

First class and higher-order functions

First-class functions is just a feature, that means all functions are values. That`s it. That`s just a concept.

Higher-order are possible because the language supports first-class functions.

First-class functions

  • JavaScript treats functions as first-class citizens

  • This means that functions are simply values

  • Functions are just another "type" of object.

Store functions in variables or properties:

                                
const add = (a, b) => a + b;

const counter = {
  value: 23,
  inc: function () {
    this.value++;
  },
};

Pass functions as arguments to other functions: 

                                
const greet = () => console.log('Hey Artem');
document.querySelector('.buy').addEventListener('click', greet);

Also we can:

  • Return functions from functions, and call methods on function

  • Call methods on functions

                                
counter.inc.bind(someOtherObject);

Higher-order functions

Higher-order functions - its a function that recieves another function as an argument, that returns a new function, or both. This is only possible because of first-class functions.

addEventListener - higher-order function, callback function - greet: 

                                
document.querySelector('.buy').addEventListener('click', greet);

Function that return new function:

                                
function count() {
  let counter = 0;

  return function () {
    counter++;
  };
};

Callback functions

Callback functions help us write more absctract code (level of abstraction). 

                                
const oneWord = function (str) {
  return str.replace(/ /g, '').toLowerCase();
};

const upperFirstWord = function (str) {
  const [first, ...others] = str.split(' ');
  return [first.toUpperCase(), ...others].join(' ');
};

Higher-order function - operating on a higher-level of abstraction, leaving the low-level details to other low-level functions.

Deligating the string transform to other lower-level functions witch are oneWord and upperFirstWord. 

                                
const transfromer = function (str, fn) {
  console.log(`Transformed string: ${fn(str)}`);
  console.log(`Transformed by: ${fn.name}`);
};

transfromer('JavaScript is the best!', upperFirstWord);
transfromer('JavaScript is the best!', oneWord);

JavaScript uses callbacks all the time:

                                
const high5 = function () {
  console.log('👋');
};
document.body.addEventListener('click', high5);

Array callback function:

                                
['Artem', 'Nika', 'Malta', 'Toki'].forEach(high5);

Call and apply methods

Call and apply methods do the same, calling functions in a different this context. But call method waiting for arguments passed in string and apply in array. But from ES6 we can use spread operator to pass values from array like a string in call method. So apply not used anymore.

Call and apply fiddle

Using call and apply:

                                
book.call(this, value, value);
book.call(swiss, 406, 'Tom Willer');

book.apply(this, [value, value])
book.apply(swiss, [flightData]);

book.call(swiss, ...flightData);

Bind method

Bind method is like a call method, but we not calling now, but waiting for the execution.

Where to use call and bind:

  • Do I need to store this function in a variable for later use? If so, use bind()

  • Do I need to pass this function as an argument? If so, use bind()

  • Do I need to call this function right now? If so, use call().

Even listeners:

                                
const lufthansa = {
  airline: 'Lufthansa'
}

lufthansa.planes = 300;
lufthansa.buyPlane = function () {
  this.planes++;
};

// error, this keyword looks at button with .buy class
document.querySelector('.buy').addEventListener('click', lufthansa.buyPlane);

// right this keyword, that looks at luftsansa object
document.querySelector('.buy').addEventListener('click', lufthansa.buyPlane.bind(lufthansa));

Partial application:

It allows us to create new functions by fixing some of the arguments of an existing function, while leaving others open for later use. 

                                
const addTax = (rate, value) => value + value * rate;
console.log(addTax(0.1, 200));

const addVAT = addTax.bind(null, 0.23); // skip 1 parameter (this value) and set tax-rate to 0.23
console.log(addVAT(100));

Immediately invoked function expressions (IIFE)

IIFE - a JavaScript function that runs as soon as it is defined.

We have no access to variables in function scope (execution context / incapsulated). So IIFE help us to write isolated variables for functions.

We can also use {bracket notation}, because it`s also create an isolated scope in ES6, unless we want to use var 🥲.

But if we need to execute function just once, then the IIFE a good choice.

IIFE example:

                                
(function () {
  console.log('This will never run again');
  const isPrivateFunc = 21;
})();

(() => console.log('This will never run again'))();

Block scope example:

                                
{
  const isPrivateBlock = 21;
  var notPrivate = 44;
}

Closures

A closure is the combination of a function bundled together (enclosed) with references to its surrounding state (the lexical environment). In other words, a closure gives you access to an outer function's scope from an inner function. In JavaScript, closures are created every time a function is created, at function creation time.

  • Any function always has access to the variable enviroment of the execution context in which the function was created

  • A function has access to the variable enviroment (VE) of the execution context in which it was created

  • Closure: VE attached to the function, exactly as it was at the time and place the function was created

  • When execution context was destroyed, VE still living somewhere in the JS Engine

  • Closure priority > scope chain priority.

                                
const secureBooking = function () {
  let passengerCount = 0;

  return function () {
    passengerCount++;
    console.log(`${passengerCount} passengers}`);
  };
};

const booker = secureBooking(); // store returned function to a variable booker

booker(); // 1 passangers
booker(); // 2 passangers
booker(); // 3 passangers

  • A closure is the closed over variable enviroment of the execution context in which a function was created, even after that execution context is gone

  • Less formal: A closure gives a function access to all the variables of its parent function, even after that parent function has returned. The function keeps a reference to its outer scope, which preserves the scope chain throughout time

  • Less formal: A closure makes sure that a function doesn't loose connection to variables that existed at the function's birth place

  • Less formal: A closure is like a backpack that a function carries around wherever it goes. This backpack has all the variables that were present in the enviroment where the function was created

  • The main advantage of closures is that they help us create private variables and methods, preventing other parts of the code from accidentally modifying or accessing them.

We do not have to manually create closures, this is a JavaScript feature that happens automatically. We can't even access closed-over variables explicitly. A closure is not a tangible JavaScript object.
