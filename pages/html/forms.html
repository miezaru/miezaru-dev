Based on the element name and the type attribute, browsers show different user interfaces, use different validation rules, and provide many other features. Using the appropriate form control helps you build better forms. 

                                
<label for="color">What is your favorite color?</label>
<input type="text" id="color" name="color">

The for attribute on the label element matches the id attribute on the input.

Input types

There are different types of form controls, all with useful built-in features that work across different browsers and platforms.

Default value

The default value for type is text. This means that if you simply want a text <input>, you can leave out the type attribute.

Multiple lines of text

                                
<label for="comment">Comment</label>
<textarea id="comment" name="comment"></textarea>

List of options

                                
<label for="color">Color</label>
<select id="color" name="color">
  <option value="orange">Orange</option>
  <option value="pink">Pink</option>
</select>

Grouping form controls

                                
<fieldset>
  <legend>What is your favorite web technology</legend>

  <label for="html">HTML</label>
  <input type="radio" name="webfeature" value="html" id="html">

  <label for="css">CSS</label>
  <input type="radio" name="webfeature" value="css" id="css">
</fieldset>

Submitting a form

                                
<button>Submit</button>
<input type="submit" value="Submit">

Every <button> element inside a form works as a Submit button by default. To disable the default Submit behavior, you can add type="button" to the <button>.

Help users avoid re-entering data

Autocomplete

Make it more convenient for users to fill out forms with HTML Autocomplete.

Based on this information, browsers can offer the option to autofill a field with previously entered data of the same type. Browsers can even offer to autofill an entire form.

Use sensible attribute values

Use email as a value for the name, id, and type attribute. Three hints for the browser that this is an email field. 

                                
<label for="email">Email</label>
<input type="email" name="email" id="email">

The autocomplete attribute 

                                
<label for="name">Name</label>
<input type="text" name="name" id="name" autocomplete="name">

Help users enter the right data

Required

The required attribute tells the browser that the field is mandatory. The browser also tests if the entered data matches the format of the type. 

                                
<label for="name">Name (required)</label>
<input required type="text" id="name" name="name">

Minlenght

                                
<label for="password">Password (required)</label>
<input required minlength="8" type="password" id="password" name="password">

For numerical input types use min and max to achieve the same result.

Advanced validation rules

Sometimes you want to define more advanced validation rules. Use HTML pattern attribute, and you can define a regular expression as the value. 

                                
<label for="animal">What is your favorite animal? (required)</label>
<input required pattern="[a-z]{2,20}" type="text" id="animal" name="animal">

Here, only lowercase letters are allowed; the user has to enter at least two characters, and not more than twenty

Communicate your validation rules

Make sure all users understand your validation rules. For this, connect the form control with an element that explains the rules. To do so, add an aria-describedby attribute to the element with the id of the form. 

                                
<label for="password">Password (required)</label>
<input required minlength="8" type="password" id="password" name="password" aria-describedby="password-minlength">
<div id="password-minlength">Enter at least eight characters</div>

Styling web forms

:invalid pseudo-class to add styles to invalid form controls. Also the :valid pseudo-class for styling valid form elements 

                                
input:required {}      // required
input:invalid {}       // invalid
input:user-invalid {}  // user-invalid

Radio buttons :default (checked) :checked (selected)

Sometimes designers color the invalid state red, using the :invalid pseudo-class. However, to communicate error or success you should never rely only on color. For people with red-green color blindness a green and a red border look almost the same. It's impossible for them to see if there was an error or success. Add text or an icon to make it obvious.

In practice :invalid is tricky to work with. Invalid form fields are already marked as :invalid before user interaction, which may confuse users. The :user-invalid pseudo-class solves this issue, as the styles are only applied after user interaction.

Validation with JavaScript

Provide meaningful error messages

Error messages are not identical in every browser. To achieve same message to everyone, use the setCustomValidity() method of the Constraint Validation API. 

                                
const nameInput = document.querySelector('[name="name"]');

nameInput.addEventListener('invalid', () => {
    nameInput.setCustomValidity('Please enter your name.');
});

Add real-time validation in JavaScript by listening to the onblur event of a form control, and validate the input immediately when a user leaves a form field.

Design recommendation

Use a label for every form control

Label wording

                                
<label for="email">Enter your email address</label>
<label for="email">Email address</label> // better

Best label position - TOP

Make forms big enought. The recommended tap target size of a button is at least 48px.

Show errors where they happen

Make it clear to users what data to enter

                                
<label for="password">Password (required)</label>
<input required minlength="8" type="password" id="password" name="password" aria-describedby="password-minlength">
<span id="password-minlength">Enter at least eight characters</span>

Make it clear to users which fields are required

                                
<label for="name">Name (required)</label>
<input name="name" id="name" required>
